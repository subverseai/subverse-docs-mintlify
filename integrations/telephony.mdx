## Outbound trunk
Setup your voice agent to make outgoing calls from your number.
### Create a SIP trunk
1. Buy a virtual number from your cloud telephony service provider.
2. There are 2 methods for authentication: IP whitelisting, or auth credentials.
3. Currently SubVerse SIP server does not have a static IP, so if you opt for the IP route, you need to whitelist all the IP addresses in the range like this: `0.0.0.0/1`, `128.0.0.0/1`.
4. It is recommended that you also setup Auth Username and Password for secure communication.
5. For detailed provider-specific instructions, see our configuration guides for [Twilio](/integrations/telephony-twilio), [Telnyx](/integrations/telephony-telnyx), or [Plivo](/integrations/telephony-plivo).

### Link to the voice agent

<img
  style={{ borderRadius: '0.5rem', height: '300px' }}
  src="/images/integrations/telephony/link-outbound-trunk.png"
/>

1. Provide a **Name** for the connection.
2. Input the **Phone Number** with country code.
3. Select your **SIP Credentials** from the dropdown menu. These credentials contain the SIP address and authentication details needed for the trunk. 
   - [Learn how to create SIP credentials](/credentials/types/sip-credentials)
   - All credential fields are **encrypted** for security
4. You can start using this number for making calls through voice agents interface, workflows, or call APIs.

## Inbound trunk
Setup your voice agent to accept incoming calls on your number.
### Create a SIP trunk
1. Buy a virtual number from your cloud telephony service provider.
2. Get your **Origination URI** from the SubVerse Telephony dashboard (it will look like: `sip:xxxxx.sip.livekit.cloud`).
3. Configure an inbound SIP trunk in your provider's dashboard using the Origination URI from SubVerse.
4. It is recommended that you also setup Auth Username and Password for secure communication.
5. For detailed provider-specific instructions, see our configuration guides for [Twilio](/integrations/telephony-twilio), [Telnyx](/integrations/telephony-telnyx), or [Plivo](/integrations/telephony-plivo).

<Tip>
  Use `xxxxxxxxx.india.sip.livekit.cloud` for region pinning to India SIP servers.
</Tip>

### Link to the voice agent

<img
  style={{ borderRadius: '0.5rem', height: '250px' }}
  src="/images/integrations/telephony/link-inbound-trunk.png"
/>

1. Provide a **Name** for the connection.
2. Input the **Phone Number** with country code.
3. Select your **SIP Credentials** from the dropdown menu. These credentials contain the authentication details needed for the trunk.
   - [Learn how to create SIP credentials](/credentials/types/sip-credentials)
   - All credential fields are **encrypted** for security
4. Configure voice agent mapping (optional):
   - **Agent**: Select the voice agent you want to use while accepting calls
   - **Agent Version**: Choose version number or use "default"/"draft" (defaults to "default")

<Tip>
  Every time you remove and again map the voice agent use case, make sure to map it again to the number to ensure it has the updated context and settings.
</Tip>

## Troubleshooting

### Credential Selection Issues

**No SIP credentials appear in dropdown**
- Ensure you have [created SIP credentials](/credentials/types/sip-credentials) first
- Check that your credentials have been saved successfully
- Verify you have the necessary permissions to access credentials

**Selected credentials not working**
- Test your SIP credentials in the [Credentials section](/credentials/overview)
- Verify the SIP address and authentication details are correct
- Check if your SIP provider account is active and properly configured

**Connection failed after selecting credentials**
- Verify network connectivity to your SIP provider
- Check firewall settings allow SIP traffic (ports 5060/5061)
- Ensure your SIP provider supports the authentication method used
- Contact your SIP provider if the issue persists

**Call quality issues**
- Check your network bandwidth and latency
- Verify codec compatibility with your SIP provider
- Test with different audio quality settings in your voice agent configuration
